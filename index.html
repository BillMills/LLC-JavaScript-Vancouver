Variables How-to

Declare variables with the keyword var . 

var email

Assign a values to the variable using the equals (=) sign.
End the statement with a semicolon (;). 

var email = "hello@christinatruong.com";

Now the computer knows my email address!

Statements are commands that tell the computer what to do. Ending the statement with a semicolon is like ending a sentence with a period. We are telling the computer that the statement is done.

Various Variables

Different types of variables hold different kinds of values.

1. Numeric variable (integers & decimals)

var someIntegerNumber = 10;
var someDecimalNumber = 10.5;

2. String variable (letters, words & sentences)

var singleWordString = "hello";
var sentenceString = "Hello, good day to you!";
var numberString = "10";
Note: string values are always written inside double quotes(")

3. Boolean variable (true/false)

var isSaturday = false;
var isSunday = true;

Variables: Tips & Best Practices

Variables cannot start with a number or any special characters except for underscores (_) and dollar signs ($).

Variables: Tips & Best Practices

Variables cannot contain spaces. Use a convention called camel case in which every new word is capitalized. It's called camel case because it looks likeCamelHumps

JavaScript is case sensitive so variable names are also case sensitive. These two variable names below are not the same. 

var camelCaseVariable = "Hello";
var camelcasevariable = "Goodbye";

Another convention is using underscores to separate words. This is common in other programming languages (like PHP). 

var spread_out_words = "Hello";

It is important to choose one method and be consistent throughout the entire project.

Variables: Tips & Best Practices

When naming a variable, it's best to give it a descriptive name.

It makes it easier to see at a glance, what kind of value that variable is going to hold.

This is especially useful when sharing code with others who may not use the same kinds of abbreviations. 

var firstName = "Clear that first name will go here";
var fn = "Not as obvious as firstName";
var x = "Not clear at all.";

Conditional Statements

Declaring variables with "static" values can be boring. Programming is about generating stuff on the fly! If... else statements are conditional statements that can be used to perform different actions based on different conditions. There are 3 variations:

If Statement - executes the code only if the condition is true 

if (condition) {
  execute this code if condition is true
}

If...Else Statement - executes some code if the condition is true and different code if condition is false 

if (condition) {
    execute this code if condition is true
} else {
  execute this code if condition is not true
}

If...Else If...Else Statement - use this statement to execute different code for multiple conditions 

if (condition) {
    execute this code if the first condition is true
} else if (condition) {
  execute this code if the second condition is true
} else {
  execute this code if neither conditions are true
}

Functions & Methods

The browser executes the JavaScript as soon as the page loads. But sometimes you need to control when it executes. This is where functions come into play.

    Functions:
  • group together related lines of code and stores them into descriptive names.
  • can be reused just by calling the function name
  • created by using function keyword
  • use parameters to vary the output using different inputs
  • define the function first but it will not be executed until you call the function
  • Functions and methods are pretty much the same thing in JavaScript except that methods are associated with an object and functions are not. Basically, they make things happen. (We will be talking about objects in greater detail later on.)

Defining and Calling a Function

There are different ways to define a function. Below is the most common way of writing a function. (We'll talk about another way later!)

Function syntax: 
function nameOfFunction() {
  //Instructions here
}

Functions and Parameters

Using parameters makes functions more flexible. Functions can be used with one parameter, more than one parameter or none at all. When using a function without parameters, you can't change the output by changing the parameters in the function call.
